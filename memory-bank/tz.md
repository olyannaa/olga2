# ТЗ ISET Group (актуальная реализация)

## Роли
- Администратор (`admin`) с возможным скоупом по подразделениям и флагом `canApproveSubcontracts`.
- ГИП (`gip`) — создание и управление своими проектами.
- Исполнитель (`executor`) — работа с задачами и таймшитами.
- Бухгалтер (`accountant`) — таймшиты всех сотрудников и чтение проектов.

## Модули
- Проекты: карточка, статусы, бюджет, архивирование/восстановление.
- WBS: разделы до 3 уровней, авто‑задачи.
- Задачи: проектные, личные, бухгалтерские, субподряды, статусная история.
- Субподряды: создание по разделам, согласование админами по подразделениям.
- Таймшиты: месяц/неделя, day‑off коды б/к/о/в.
- Организация: подразделения, сотрудники, контрагенты (UI для admin/accountant).
- Аналитика проекта: прогресс, трудозатраты, стоимость.
- Чат проекта: текстовые сообщения.

## Важные правила
- Нет саморегистрации.
- Субподряд доступен исполнителю только после `approved`.
- Задачи WBS редактируются через разделы, а не напрямую.
- Файлы — метаданные/URL; реальное хранилище не интегрировано.
